Searching

see searching docs

A regular search may be followed by 1 or more of -> followed by a client filter

Client Filters

After a regular search, as many -> can be used to apply client filters. The basic client filters include:

  • head n: get the n top items
  • tail n: get the n bottom items
  • shuf: shuffle the items
  • sort b: sort by b which can be a for alphabetical or e for js eval
  • !child: to ensure the item is not a child
  • !copy: to ensure the item is not a copy

If the filter begins and ends with a / a regex search will be performed

Url Parameters

the /ui has a few url parameters.

  • uid: the user's id to display
    This parameter or uname is required in order to know which user's items to display
  • uname: the username to display
    • q: an initial search query
  • item-id: the item id to show (same as ?q=ItemId%20=id
    • display: get rid of the ui elements, and display the first result found by q,
  • or the item specified by item-id

Include

In a display entry, include will determine what is included in the events table and total cost

Special Paths

There are a couple of paths that can be used as shortcuts

  • /user/{username}: redirects to /ui?uname={username}
  • /uid/{uid}: redirects to /ui?uid={uid}
  • /login: performs a basic www-authentication, all url parameters are forwarded to /ui
  • /settings.html: user settings editor
